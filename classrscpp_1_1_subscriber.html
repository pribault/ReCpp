Detailed Description

template<typename T>
class rscpp::Subscriber< T >

Will receive call to onSubscribe(Subscription&) once after passing an instance of rscpp::Subscriber to rscpp::Publisher::subscribe(Subscriber&).

No further notifications will be received until rscpp::Subscription::request(size_t) is called.

After signaling demand:

  • One or more invocations of onNext up to the maximum number defined by rscpp::Subscription::request(size_t)
  • Single invocation of onError(const std::exception_ptr&) or rscpp::Subscriber::onComplete() which signals a terminal state after which no further events will be sent.

Demand can be signaled via rscpp::Subscription::request(size_t) whenever the rscpp::Subscriber instance is capable of handling more.

virtual void rscpp::Subscriber< T >::onSubscribe ( Subscription &  subscription)
virtual

Invoked after calling rscpp::Publisher::subscribe(Subscriber&).

No data will start flowing until rscpp::Subscription::request(size_t) is invoked.

It is the responsibility of this rscpp::Subscriber instance to call rscpp::Subscription::request(size_t) whenever more data is wanted.

The rscpp::Publisher will send notifications only in response to rscpp::Subscription::request(size_t).
